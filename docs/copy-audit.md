# Копирайтерский аудит stroistroy.ru

**Дата:** 2025-06-05  
**Цель:** коротко, чётко, польза клиенту — без воды и «строительного пафоса».

---

## Принципы правок

1. **Один экран — одна мысль.** Hero и блоки отвечают на вопрос «что я получу?», не «кто мы такие».
2. **CTA = действие + результат:** «Получить расчёт», «Подобрать проект», не «Узнать больше».
3. **Цена всегда с оговоркой:** «ориентир», «после вводных» — снимает недоверие и снижает отказы.
4. **Боль → решение в одном предложении**, без повторов между блоками.
5. **SEO-тексты — 2 абзаца max**, без keyword stuffing.

## Карта смыслов (единый источник)

```
positioning.ts
├── offers.primary → hero headline
├── pageCopy.* → страницы
├── cta.* → все кнопки
├── clientSegments → audience на главной
└── homeScenarios → карточки сценариев

copy.ts / home.ts / *-copy.ts → UI-блоки страниц
```

---

## Что ещё можно улучшить (не блокер MVP)

- **Кейсы:** заменить заглушки на реальные объекты + отзывы (контент, не код).
- **Блог:** статьи не проходили этот аудит — отдельная итерация.
- **Карточки проектов:** `projectDescriptionDefault` — шаблон; лучше уникальные описания в CMS.
- **A/B:** `offers.variants` — для рекламы и лендингов, не на сайте.
