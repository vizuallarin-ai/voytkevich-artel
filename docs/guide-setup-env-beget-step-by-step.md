# Часть 0. Как вообще зайти на сервер и открыть `.env`

## Способ 1 — через панель Beget (если SSH с Windows не работает) ⭐

1. Откройте браузер → [https://beget.com](https://beget.com) → **Войти**.
2. Меню **«Облако»** или **«VPS»** → выберите ваш VPS (`159.194.212.82`).
3. Найдите кнопку **«Терминал»**, **«Консоль»**, **«SSH в браузере»** или **VNC** — откроется чёрное окно с текстом.
4. Вы должны увидеть что-то вроде `root@...:~#` — это командная строка Linux.

Дальше все команды ниже вводите **в это окно** и нажимайте **Enter**.

## Способ 2 — SSH с компьютера

1. Откройте **PowerShell** или **Windows Terminal**.
2. Введите:

```text
ssh root@159.194.212.82
```

3. Пароль — из Beget → VPS → **Пароль root** (при вводе символы не видны — это нормально).
4. Если пишет `Permission denied` — используйте **Способ 1** (терминал в браузере Beget).

## Как открыть файл `.env` для редактирования

### Шаг 1 — перейти в папку проекта

Скопируйте и вставьте в терминал (правая кнопка мыши часто вставляет):

```bash
cd /opt/stroistroy
```

Нажмите **Enter**.

### Шаг 2 — проверить, что файл есть

```bash
ls -la .env
```

Должно показать файл `.env`. Если **«No such file»**:

```bash
cp .env.example .env
```

### Шаг 3 — открыть редактор

```bash
nano .env
```

Откроется текстовый редактор прямо в терминале.

### Как пользоваться nano (важно)

| Действие | Как |
|----------|-----|
| Листать | Стрелки ↑ ↓ |
| Печатать | Просто печатайте |
| Сохранить | **Ctrl+O**, затем **Enter** |
| Выйти | **Ctrl+X** |

Внизу экрана nano подсказки: `^O` = Ctrl+O, `^X` = Ctrl+X.

### Как выглядит строка в `.env`

```env
ИМЯ_ПЕРЕМЕННОЙ=значение
```

- **Без пробелов** вокруг `=`
- **Без кавычек** (обычно)
- Одна переменная — одна строка
- Строки с `#` в начале — комментарии, сервер их игнорирует

**Пример — было:**

```env
NEXT_PUBLIC_YM_ID=
```

**Стало (подставьте свой номер):**

```env
NEXT_PUBLIC_YM_ID=98765432
```

# Часть 1. Яндекс.Метрика

## 1.1. Создать счётчик (на сайте Яндекса, не на сервере)

1. Браузер → [https://metrika.yandex.ru](https://metrika.yandex.ru)
2. Войти под Яндекс-аккаунтом (тот, что будет смотреть статистику).
3. Кнопка **«Добавить счётчик»**.
4. Заполнить:
   - **Имя:** `stroistroy.ru`
   - **Адрес сайта:** `https://stroistroy.ru`
   - Часовой пояс: **Иркутск** (UTC+8) или ваш
   - Вебвизор — по желанию (можно включить)
5. **«Создать счётчик»**.
6. На странице счётчика найдите **номер** — только цифры, например `104382561`.  
   **Скопируйте его** в блокнот.

> Код счётчика вручную вставлять на сайт **не нужно** — это уже сделано в проекте. Нужен только номер в `.env`.

## 1.2. Вписать номер в `.env` на сервере

1. Терминал Beget (см. Часть 0).
2. `cd /opt/stroistroy`
3. `nano .env`
4. Найдите строку (листайте стрелками):

```env
NEXT_PUBLIC_YM_ID=
```

5. После `=` впишите номер **без пробелов**:

```env
NEXT_PUBLIC_YM_ID=104382561
```

6. Проверьте, что есть строка (если нет — добавьте):

```env
NEXT_PUBLIC_SITE_URL=https://stroistroy.ru
```

7. **Ctrl+O** → **Enter** → **Ctrl+X**.

## 1.3. Пересобрать сайт на сервере

```bash
cd /opt/stroistroy
docker compose up -d --build
```

## 1.4. Проверить у себя на компьютере

1. Откройте `https://stroistroy.ru`
2. **F12** → вкладка **Network** (Сеть)
3. В фильтре введите `metrika`
4. Должна быть загрузка `tag.js` с `mc.yandex.ru`

Или: правой кнопкой на странице → **Просмотр кода страницы** → Ctrl+F → ищите `mc.yandex.ru`.

## 1.5. Цели в Метрике (чтобы видеть заявки)

1. [metrika.yandex.ru](https://metrika.yandex.ru) → ваш счётчик
2. Слева **«Настройки»** → **«Цели»**
3. **«Добавить цель»**
4. Тип: **«JavaScript-событие»**
5. **Идентификатор:** `calculator_submit` (для калькулятора)
6. Сохранить.

Повторите для (по желанию): `form_submit`, `project_request`.

После тестовой заявки с `/calculator` цель появится в отчётах через несколько минут.

# Часть 2. Яндекс.Вебмастер

## 2.1. Добавить сайт

1. [https://webmaster.yandex.ru](https://webmaster.yandex.ru)
2. **«Добавить сайт»**
3. URL: `https://stroistroy.ru`
4. **«Добавить»**

## 2.2. Подтвердить, что сайт ваш (способ через DNS — без правок кода)

1. Вебмастер предложит способы → выберите **«DNS-запись»** / **«Мета-тег»** / **«HTML-файл»**.  
   **Рекомендуем DNS** — не трогаем код.

2. Для **DNS** Вебмастер покажет строку, например:

```text
yandex-verification: a1b2c3d4e5f6g7h8
```

3. Откройте **новую вкладку** → [beget.com](https://beget.com) → войти.
4. **Домены и поддомены** → `stroistroy.ru` → **DNS** / **Управление DNS**.
5. **Добавить запись**:
   - **Тип:** `TXT`
   - **Имя / Subdomain:** `@` или оставить пустым (главный домен)
   - **Значение / Value:** вся строка от Яндекса целиком (`yandex-verification: ...`)
   - **TTL:** 300 или по умолчанию
6. **Сохранить**.
7. Подождать **10–60 минут** (иногда до 24 ч).
8. Вернитесь в Вебмастер → **«Проверить»**.

## 2.3. Отправить карту сайта (sitemap)

1. Вебмастер → ваш сайт → **«Индексирование»** → **«Файлы Sitemap»**
2. Поле URL → вставьте:

```text
https://stroistroy.ru/sitemap.xml
```

3. **«Добавить»**.

⚠️ **Если статус «ошибка»** — на prod sitemap может отдавать 500. Тогда напишите в Cursor «почини sitemap» — без рабочего sitemap Вебмастер карту не примет.

## 2.4. Связать с Метрикой

1. Вебмастер → **«Настройки»** → **«Связь с Яндекс.Метрикой»**
2. Выбрать счётчик из Части 1 → сохранить.

# Часть 3. Telegram — уведомления о заявках

Заявка с сайта → сообщение в Telegram + запись в CRM.

## 3.1. Создать бота

1. На телефоне или в Telegram Desktop откройте [@BotFather](https://t.me/BotFather)
2. Отправьте команду:

```text
/newbot
```

3. BotFather спросит **имя** — напишите, например:

```text
Stroistroy Leads
```

4. Спросит **username** — лatin, на конце `bot`, например:

```text
stroistroy_leads_bot
```

5. BotFather пришлёт **токен** — длинная строка вида:

```text
7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**Скопируйте целиком** в блокнот. Это `TELEGRAM_BOT_TOKEN`.

## 3.2. Узнать chat_id (куда слать сообщения)

### Вариант А — себе в личку

1. В Telegram найдите **вашего нового бота** по username → **Start** / **Запустить**.
2. Напишите боту что угодно, например `/start`.
3. На компьютере откройте в браузере (подставьте **свой** токен вместо `ТОКЕН`):

```text
https://api.telegram.org/botТОКЕН/getUpdates
```

Пример (не ваш, формат такой):

```text
https://api.telegram.org/bot7123456789:AAHxxx/getUpdates
```

4. В ответе JSON найдите:

```json
"chat":{"id":123456789
```

Число `123456789` — это **TELEGRAM_CHAT_ID**.

### Вариант Б — группа менеджеров

1. Создайте группу в Telegram.
2. Добавьте туда бота (через поиск по username).
3. Напишите в группе любое сообщение.
4. Снова откройте `getUpdates` в браузере.
5. `chat.id` будет **отрицательным**, например `-1001234567890` — это тоже нормально.

## 3.3. Вписать в `.env` на сервере

1. Терминал Beget → `cd /opt/stroistroy` → `nano .env`
2. Найдите или добавьте строки:

```env
TELEGRAM_BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_CHAT_ID=123456789
```

3. **Ctrl+O** → **Enter** → **Ctrl+X**

## 3.4. Перезапустить (без --build)

```bash
cd /opt/stroistroy
docker compose up -d
```

## 3.5. Проверка

1. Откройте `https://stroistroy.ru/calculator`
2. Отправьте **тестовую заявку** (можно свой телефон).
3. Должно прийти сообщение в Telegram.
4. Зайдите в CRM: `https://stroistroy.ru/dashboard/login` (пароль — `DASHBOARD_ACCESS_TOKEN` из `.env`).

# Часть 4. Email (SMTP) — опционально

Нужно, если хотите **письма на почту** в дополнение к Telegram или вместо него.

## 4.1. Почта на домене (Beget + Yandex)

1. [beget.com](https://beget.com) → **Домены** → `stroistroy.ru` → раздел **«Почта»**.
2. Подключите почту на домене (Yandex 360 / инструкция Beget для MX-записей).
3. Создайте ящик, например `info@stroistroy.ru`.
4. Запомните пароль от ящика.

## 4.2. Пароль приложения для SMTP (не обычный пароль!)

1. [https://id.yandex.ru](https://id.yandex.ru) → войти тем же аккаунтом, что почта на домене.
2. **Безопасность** → **Пароли приложений**.
3. **Создать** → тип **«Почта»** / **«Другое»**.
4. Скопировать пароль (показывается один раз).

## 4.3. Вписать в `.env` на сервере

`nano .env` → добавить или изменить:

```env
LEADS_NOTIFICATION_EMAIL=info@stroistroy.ru
EMAIL_FROM=info@stroistroy.ru
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=info@stroistroy.ru
SMTP_PASS=сюда-пароль-приложения-из-yandex
```

Сохранить: **Ctrl+O** → **Enter** → **Ctrl+X**

## 4.4. Перезапуск

```bash
cd /opt/stroistroy
docker compose up -d
```

## 4.5. Проверка

Тестовая заявка с сайта → письмо на `info@stroistroy.ru` (проверьте «Спам»).

# Часть 6. Частые ошибки

| Что случилось | Что делать |
|---------------|------------|
| `nano: command not found` | `apt install nano` или используйте редактор файлов в панели Beget |
| Метрика не появилась | После правки `NEXT_PUBLIC_YM_ID` обязательно `docker compose up -d --build` |
| Telegram молчит | Боту написали `/start`? Верный `chat_id` из getUpdates? |
| SMTP «auth failed» | Нужен **пароль приложения**, не пароль от входа в почту |
| Забыли пароль CRM | `nano .env` → строка `DASHBOARD_ACCESS_TOKEN` |
| Не знаете токен CRM | В `.env` на сервере; на GitHub его нет |
